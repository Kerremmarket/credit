📦 BACKEND SERVICE (API-LITE)
─────────────────────────────────────────────────────────────────
  Root Directory:  backend/
  Build Arg:       INCLUDE_TF = false
  
  Environment Variables:
  ├─ PORT = 8000
  ├─ FRONTEND_ORIGIN = https://your-frontend.railway.app
  └─ DEMO_MODE = true
  
  Volume:
  └─ Mount: /data → Upload cs-training-sample-small.csv

─────────────────────────────────────────────────────────────────

🌐 FRONTEND SERVICE
─────────────────────────────────────────────────────────────────
  Root Directory:  frontend/
  
  Environment Variables:
  └─ VITE_API_BASE = https://your-backend.railway.app

─────────────────────────────────────────────────────────────────

✅ VERIFICATION STEPS
─────────────────────────────────────────────────────────────────
  1. Health check:
     GET https://your-backend.railway.app/api/health
     → Should return: {"status":"ok","demo_mode":true}

  2. Open frontend:
     https://your-frontend.railway.app
     → Should load UI without errors

  3. Test training:
     → Select features → Train Logistic or XGBoost
     → (MLP won't work unless INCLUDE_TF=true)

─────────────────────────────────────────────────────────────────

🔧 TROUBLESHOOTING
─────────────────────────────────────────────────────────────────
  CORS Error?
  → Check FRONTEND_ORIGIN matches exact frontend URL (with https://)
  
  Data file not found?
  → Use path: /data/cs-training-sample-small.csv in UI
  
  MLP fails?
  → You built with INCLUDE_TF=false (saves 900 MB!)
  → Rebuild with INCLUDE_TF=true if you need neural networks

─────────────────────────────────────────────────────────────────

📊 SIZE COMPARISON
─────────────────────────────────────────────────────────────────
  Backend (INCLUDE_TF=false):  ~300 MB  ← Recommended
  Backend (INCLUDE_TF=true):   ~1.2 GB
  Frontend:                    ~50 MB
  Data volume:                 ~20 MB

─────────────────────────────────────────────────────────────────

🚀 DEPLOYMENT ORDER
─────────────────────────────────────────────────────────────────
  1. Deploy Backend → Get URL
  2. Deploy Frontend → Set VITE_API_BASE to backend URL
  3. Update Backend FRONTEND_ORIGIN to frontend URL
  4. Test everything!
